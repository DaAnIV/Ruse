📝 Example Usage

You can input fields in JSON format. The parser will filter out certain fields (message, panic.backtrace, panic.location) and extensions (.mermaid, .json) just like in the main log viewer.

Example Input:

{ "user_id": "12345", "action": "create_post", "timestamp": "2024-01-15T10:30:00Z", "ip_address": "192.168.1.100", "status": "success", "post_id": "abc123", "message": "This will be filtered out", "panic.backtrace": "This will also be filtered out", "data.mermaid": "This extension will be filtered out" }

Expected Output:

The parser will display only the non-filtered fields in a clean, structured format similar to how they appear in the main log viewer.
